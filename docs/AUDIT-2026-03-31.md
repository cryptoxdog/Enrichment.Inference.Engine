## 3. Critical Execution Flows

### Flow A: Single-Pass Enrichment (`handle_enrich`)

1. `EnrichRequest.model_validate(payload)` вҖ” Pydantic v2 validation
2. `enrich_entity(request, settings, kb_resolver, idem_store)` called
3. Idempotency check against Redis (key = `request.idempotency_key`)
4. `kb_resolver.resolve(kb_context, entity, max_fragments=3)` вҶ’ KB fragments
5. `compute_uncertainty(entity, target_schema, max_variations)` вҶ’ variation count (1вҖ“10)
6. `build_prompt(entity, object_type, objective, schema, kb_context_text, model, sonar_config)` вҶ’ payload dict
7. `asyncio.gather(*[query_perplexity(...) for _ in range(variation_count)])` via `Semaphore(max_concurrent_variations)`
8. `_process_variation_results()` вҶ’ valid list, raw\_payloads, errors, total\_tokens
9. `synthesize(valid, consensus_threshold, total_attempted)` вҶ’ fields + per\_field\_confidence
10. `EnrichResponse(fields, confidence, feature_vector, ...)` assembled and returned

### Flow B: Multi-Pass Convergence (`handle_converge`)

1. `run_convergence_loop(request, settings, kb_resolver, ...)` invoked
2. `ConvergenceState` initialized; `state.known_fields` seeded from `request.entity`
3. For each pass (1 вҶ’ `max_passes=3`):
   - Budget check via `CostTracker.can_continue()`
   - `MetaPromptPlanner.plan_pass(entity_state, pass_context, field_confidences)` вҶ’ `PromptPlan(mode, priority_fields, variation_count)`
   - Pass modes: `"discovery"` (Pass 1) вҶ’ `"targeted"` (Pass 2) вҶ’ `"verification"` (Pass 3)
   - `resolve_from_classification(mode, priority_fields, signals, domain_classification)` вҶ’ `SonarConfig`
   - `enrich_entity(pass_request, ..., sonar_config=sonar_config)` called
   - Enriched fields merged into `state.known_fields` + `state.confidence_map`
   - `ConfidenceTracker.update_field(name, value, confidence, pass_num, source)` for each field
   - `InferenceBridge.run(entity=state.known_fields, confidence_map)` вҶ’ derived fields
   - Convergence check: `confidence_tracker.has_converged()` OR `had_meaningful_improvement()` вҶ’ break
4. `_assemble_convergence_response()` builds final `EnrichResponse` with `pass_count`, `feature_vector.cost_summary`, `feature_vector.confidence_tracking`

### Flow C: Writeback to Odoo (`handle_writeback`)

1. `WriteBackOrchestrator(crm_type=CRMType.odoo, credentials, mapping_path)` instantiated
2. `orchestrator.async_write_back(domain, canonical)` called
3. Odoo XML-RPC with field name mapping from `config/crm/odoo_mapping.yaml`
4. Returns `{success, record_id, fields_written, error}`

### Flow D: Inference-Only (no search)

- When `SonarConfig.disable_search=True` (all target fields are `INFERRABLE`), `enrich_entity()` returns a zero-token `EnrichResponse` with `state="completed"` immediately
- `InferenceBridge` then fires its YAML rules against known entity fields to produce derived values

## 5. Scoring / Reasoning Mechanisms

### Consensus Engine (`app/services/consensus_engine.py`)

- Takes N validated variation responses
- Per-field: counts how many variations agree on a value
- Agreement ratio = `agreements / total_attempted`
- Field accepted if ratio вүҘ `consensus_threshold` (default 0.65)
- `per_field_confidence` returned for each accepted field
- Weighted by variation count and agreement density

### Search Optimizer Scoring (`app/engines/search_optimizer.py`)

- `FieldDifficulty` enum: `TRIVIAL` вҶ’ `PUBLIC` вҶ’ `FINDABLE` вҶ’ `OBSCURE` вҶ’ `INFERRABLE`
- Dominant difficulty of `target_fields` drives model selection:
  - `TRIVIAL`/`PUBLIC` вҶ’ `sonar` (cheap)
  - `FINDABLE`/targeted + `gate_fields_missing > 0` вҶ’ `sonar-pro`
  - `OBSCURE` вҶ’ `sonar-pro`
  - discovery mode (Pass 1) вҶ’ `sonar-pro`
  - `ambiguous_fields вүҘ 2` overlap вҶ’ `sonar-reasoning`
- Variation count = `base(3)` + adjustments for difficulty, avg\_confidence, budget
- Cost model: `MODEL_COST_PER_1K Г— CONTEXT_COST_MULTIPLIER Г— (max_tokensГ—1.5/1000) Г— variations`

### ConfidenceTracker (`app/engines/convergence/confidence_tracker.py`)

- Tracks per-field confidence across passes
- `has_converged()` = all tracked fields above `confidence_threshold`
