Anti-Patterns to Avoid

❌ Don't test scope-creep modules (score/, health/, simulation_bridge.py)
❌ Don't write integration tests that call real APIs without @pytest.mark.integration
❌ Don't share mutable state across tests (use fixtures)
❌ Don't test implementation details (test behavior, not internal methods)
❌ Don't write tests that depend on execution order (each test must be independent)
❌ Don't mock what you own (mock external dependencies, not internal modules unless necessary)
❌ Don't write one giant test (one assertion focus per test, multiple tests per function)
